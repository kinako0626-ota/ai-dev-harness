# 修正スキルマッピング

Phase 2-3 で検出された問題カテゴリに対して、どのスキル（または直接修正）を適用するかのマッピング。

## 直接修正（スキル不要）

| 問題 | 修正内容 |
|------|---------|
| 未使用 import | 削除 |
| analyze エラー | エラー内容に応じて修正 |

### デザインシステムが定義されている場合

| 問題 | 修正内容 |
|------|---------|
| `プロジェクトのデザインシステムクラス（CLAUDE.md 参照）` 未使用（ハードコード色） | `プロジェクトのデザインシステムファイル（CLAUDE.md 参照）` に定数追加 → 参照に変更 |

### コード生成がある場合

| 問題 | 修正内容 |
|------|---------|
| コード生成の再生成漏れ | `プロジェクトのコード生成コマンド（CLAUDE.md 参照）` |

## スキル経由の修正

| 検出カテゴリ | 推奨スキル | 説明 |
|---|---|---|
| デザインシステム逸脱 | `/normalize` | デザインシステム準拠の徹底 |
| エッジケース未対応 | `/harden` | 堅牢性の向上 |
| UXコピー不明確 | `/clarify` | マイクロコピー改善 |
| パフォーマンス問題 | `/optimize` | パフォーマンス最適化 |
| レスポンシブ未対応 | `/adapt` | マルチデバイス対応 |
| 共通パターンの重複 | `/extract` | コンポーネント抽出・共通化 |
| 情報過多・複雑 | `/distill` | シンプル化 |

## デザイン調整スキル

| 検出カテゴリ | 推奨スキル | 説明 |
|---|---|---|
| UIが退屈 | `/bolder` | ビジュアル強化 |
| UIが派手すぎ | `/quieter` | トーンダウン |
| 色が単調 | `/colorize` | 戦略的カラー追加 |
| 静的で動きがない | `/animate` | アニメーション追加 |
| 機能的だが楽しくない | `/delight` | ディライト要素追加 |

## 優先度ルール

1. **MUST**: 規約違反、バグリスク → 直接修正
2. **SHOULD**: 品質向上 → スキル経由修正（ユーザー承認後）
3. **NICE**: 軽微な改善 → リストアップ（実行はユーザー判断）
